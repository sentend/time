repositories используются для взаимодействия с бд, по факту они не дожны знать реализации бд и лучше взаимодействовать с абстракцией

controller взаимодействуют с определнным фреймворком, в виде роутеров

entities - для описания сущности из бд

service - для бизнес логики

mapper - это дополнительные методы домена 

repositores
	↓
service
	↓
controller
